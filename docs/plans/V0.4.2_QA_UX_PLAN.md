# Pivotglass v0.4.2 QA/UX implementation plan

**Status:** Approved — release train in progress
**Prepared:** 2026-07-21
**Baseline:** `v0.4.1` (`272df7d`)
**Release intent:** Trustworthy live feedback, complete interactive TUI history,
evidence drill-down, tested navigation, retirement migrations, and materially
differentiated original archetype environments.

No implementation work described here should begin until this plan and its
explicit decision gates are approved.

## Review team

- **Terminal QA and accessibility:** terminal input, scrolling, viewport
  retention, help, recovery, persisted state, and PTY-level verification.
- **Cyber-analyst product UX:** live investigation feedback, attention,
  navigation, accessibility, evidence/provenance inspection, and responsive
  workflows in Pivotglass.
- **Narrative systems and character QA:** mode-by-mode voice and environment
  differentiation, accessibility, analytical boundaries, roster durability,
  banter budgets, and retirement migration.
- **Primary synthesis:** cross-interface contracts, architectural sequencing,
  release gates, and preservation of evidence/inference boundaries.

The review used the current source, tests, README, `PHILOSOPHY.md`, protected
`storyboard/` studies, and protected `reckonings/UX-team.md`. It made no product
changes.

## Executive finding

Pivotglass has a strong visual foundation. Its next release should make the
cockpit operationally trustworthy before adding more spectacle.

The highest-priority gaps are:

1. **Web feedback is not incremental.** The browser receives a plan, waits for
   one synchronous investigation request, and receives evidence only after all
   sequential probes finish.
2. **Several web instruments imply measurements they do not possess.** Reactor
   power is fixed, Probe Bay counts briefing definitions, Token Core uses a
   fixed activity percentage, and Hull/Wear reflects only the latest browser
   error.
3. **Summaries are terminal nodes.** Neither interface provides a reliable path
   from a compact finding or inference to the complete normalized evidence,
   provenance, dossier contribution, uncertainty, or pivot basis.
4. **TUI dossier telemetry is not initialized from persisted dossier state.**
   A reopened workspace can contain evidence while the HUD begins at `0/9`.
5. **The TUI scroll implementation lacks real interaction-level verification.**
   Helper tests pass, but mouse dragging, wheel delivery, PageUp/PageDown,
   focus, and terminal escape sequences are not exercised through a rendered
   prompt-toolkit application.
6. **Character theming is structurally coherent but experientially subtle.**
   Both interfaces still share almost all geometry, animation, and instrument
   behavior across modes.
7. **Drunken Master is no longer a product fit.** The explicit decision for
   v0.4.2 is retirement from selectable runtime surfaces while preserving
   history and providing deterministic migration.
8. **Bobby Hill duplicates the sarcastic-sidekick function.** The explicit
   decision for v0.4.2 is retirement, with Full Troll becoming the owned home
   for dry skepticism, provocation, and sarcastic sidekick energy.

## Product commitments

### Truthful instrumentation

- No numeric meter may appear without a named, real data source.
- Unknown measurements display `N/A`, `NOT INSTRUMENTED`, or `NOT ENGAGED`.
- Decorative effects are visibly non-semantic and never resemble evidence,
  severity, progress, confidence, damage, or resource exhaustion.
- Evidence, inference, analyst notes, character narration, and system state
  remain structurally distinguishable in every theme.

### Constant feedback without invented progress

Every investigation and probe uses one shared lifecycle:

```text
planned → queued → running → succeeded | empty | failed | skipped | cancelled
```

Feedback includes current operation, elapsed time, queue position, last
meaningful transition, and recoverability. Heartbeats indicate liveness, not a
fabricated completion percentage.

### Summaries are doors

Every compact evidence or inference card exposes a deterministic drill-down.
The detail view is produced from stored data and local logic; opening it never
requires an LLM call.

### Stable semantics, expressive worlds

Character themes may change geometry, ambient layers, motion language,
instrument vocabulary, and narration. They may not change evidence ordering,
severity, provenance, analytical conclusions, or the meaning of controls.

## Shared contracts

### Investigation event envelope

One Python-owned event model feeds both interfaces. Each event includes:

- stable investigation and event IDs;
- event class and severity;
- lifecycle state;
- tool, source, target, and target type;
- created, started, updated, and completed timestamps where applicable;
- elapsed time and queue position;
- result count and artifact IDs;
- empty/failure/cancellation reason;
- retryability and supported operator actions;
- evidence, inference, or narration classification;
- optional dossier transitions and alert metadata.

The web adapter streams or exposes a resumable event cursor. The TUI consumes
the same transitions through the existing event bus. Reconnects must suppress
duplicates and recover the authoritative current state.

### Evidence detail envelope

Every evidence card receives a short session-visible ID and links to a detail
record containing:

- STIX ID, type, value, and normalized fields;
- source module and original query target;
- retrieval timestamp;
- source URL, API/version, and response digest when actually available;
- explicit `unavailable` labels when provenance fields do not exist;
- related artifacts and relationships;
- contributing dossier slots and the rule behind each status;
- supporting or conflicting observations;
- raw response fields only when safe and scrubbed of credentials;
- justified next pivots with their evidence basis.

Inference details enumerate supporting evidence IDs. Unlinked legacy summaries
are labeled `UNLINKED SUMMARY`; the UI must not manufacture a connection.

### Attention taxonomy

Theme-independent event classes establish consistent meaning:

- `DISCOVERY`
- `CORROBORATION`
- `CONTRADICTION`
- `DOSSIER TRANSITION`
- `SOURCE FAULT`
- `OPERATOR ACTION`
- `INFERENCE`
- `SYSTEM`

Severity uses a text label, glyph/pattern, and accessible announcement in
addition to color. A persistent alert record can be acknowledged without being
deleted.

## Implementation sequence

### W-042-01 — Truthful lifecycle and instrumentation

Build the shared investigation event model and incremental delivery mechanism.
Replace every approximate web meter with measured state or an honest unavailable
label.

Deliverables:

- per-probe lifecycle and timestamps;
- incremental evidence arrival before the final probe completes;
- elapsed-time heartbeat;
- explicit stop, skip, retry, and cancellation acknowledgements;
- health derived from a current loopback API check;
- configured/available/queued source counts kept distinct;
- token use shown only from actual model usage metadata;
- partial successes retained when another source fails;
- ordering, cancellation, reconnect, and duplicate-suppression tests.

Exit criteria:

- No displayed percentage lacks a tested authority.
- Every planned probe visibly reaches a terminal state.
- The user can distinguish slow progress, empty results, failure, and a hang.

### W-042-02 — Shared evidence references and drill-down

Introduce typed feed entries and evidence detail projections without replacing
the existing workspace/STIX authorities.

Web deliverables:

- selectable evidence, inference, dossier, alert, and chart elements;
- artifact/detail drawer or pane with browser-history state;
- back/close restores the originating card, chart selection, and feed position;
- dossier cells explain status, contributing evidence, and missing support;
- Flint bars provide a keyboard-accessible data table and filtered artifact
  list.

TUI deliverables:

- stable IDs shown on truncated cards;
- explicit truncation marker plus `open <id>` action;
- keyboard-opened detail overlay;
- Escape/Back returns to the exact previous viewport;
- inference-to-evidence references and actionable diagnostic details.

Exit criteria:

- Every observed claim links to evidence or says it is unlinked.
- Raw and normalized detail is inspectable without exposing credentials.
- Opening detail invokes no network service or LLM.

### W-042-03 — Attention, unread state, and recovery

Implement the shared attention taxonomy and a persistent, inspectable alert
queue.

Deliverables:

- master caution opens filtered alerts and their originating events;
- highest unread severity and unread count appear while reviewing history;
- jump-to-next-important and return-to-reading-position actions;
- critical announcements use an accessible live region without stealing focus;
- retry, skip, continue, details, and copy-diagnostic actions are executable;
- empty results are distinct from failures;
- yield commands acknowledge receipt immediately and later report application.

Exit criteria:

- Important new evidence cannot be silently hidden by viewport preservation.
- Theme animation never competes with a critical alert.
- Acknowledgement changes alert state without erasing history.

### W-042-04 — TUI truth and scrolling contract

Make the terminal cockpit truthful on first render and prove its actual input
path.

Deliverables:

- load the persisted `DossierState` on startup and every workspace switch;
- wire slot transitions with explicit lifecycle cleanup;
- derive slot counts from the canonical snapshot, not a drifting counter;
- retain the complete session transcript in an authoritative history model;
- PageUp/PageDown moves one viewport minus context rows;
- clamp all scroll positions without discarding earlier session entries;
- provide a searchable transcript index and stable entry/evidence anchors;
- support a list/detail/return interaction: select or open a summary, inspect
  its full evidence and provenance, then return to the exact transcript row;
- keep navigation discoverable through a persistent status/key-hint region and
  context-sensitive help;
- provide durable save/export as a convenience, not as compensation for an
  artificially truncated interactive history;
- real PTY tests send PageUp/PageDown and terminal wheel sequences;
- rendered mouse tests cover track click, thumb drag, release, focus, and clamps;
- narrow-terminal layouts preserve evidence IDs and recovery actions;
- `?` and typed `help` render from one command/keybinding authority.

Interaction references:

- Treat Claude Code's transcript navigation and expandable detail behavior as
  a current usability benchmark, without copying proprietary implementation or
  distinctive trade dress.
- Draw on ELM's durable list/detail/status model and Lynx's keyboard-first link
  traversal, history, predictable return behavior, and always-available help.
- Preserve Pivotglass terminology and visual identity; these products are
  interaction references, not skins.

Exit criteria:

- Reopened workspaces show the correct nine-slot dossier state immediately.
- Mouse, trackpad, PageUp/PageDown, and laptop-safe keys visibly change rows in
  an integration test.
- New output preserves the exact top visible evidence line during review.
- Every entry from the current session remains reachable, searchable, and
  openable until the operator explicitly clears or closes that session.
- Detail-open and Back/Escape round trips restore selection, focus, and the
  exact prior viewport.

### W-042-05 — Pane navigation and accessibility

Turn web and terminal panes into navigable analytical surfaces rather than
visual regions.

Deliverables:

- persistent pane switcher at desktop and compact widths;
- searchable command palette for navigation, help, characters, and hunt
  controls;
- selected pane/artifact reflected in web URL/history state;
- focus traps and focus restoration for overlays;
- help remains available on narrow screens;
- complete keyboard and screen-reader navigation;
- `prefers-reduced-motion` plus in-product `full / reduced / off` effects;
- independent narration intensity control;
- accessible chart data and non-color status cues.

Exit criteria:

- No primary control is hidden at 320, 768, 1024, or wide desktop widths.
- Automated accessibility checks report no critical violations.
- Reduced-motion mode loses no operational meaning.

### W-042-06 — Character contract and roster retirements

Extend the canonical mode definition with presentation capabilities rather than
adding parallel web-only theme logic.

Character contract dimensions:

- geometry family;
- ambient layer;
- motion language;
- instrument vocabulary;
- event flourish hooks;
- voice policy and repetition budget;
- procedural music palette and activity response;
- accessibility/reduced-motion alternative.

Drunken Master retirement:

1. Remove `drunken_master` from selectable web/TUI catalogues, completion, and
   active-roster counts.
2. Preserve immutable legacy metadata: identifier, former display name, and
   `retired_in="0.4.2"`; it carries no executable persona prompt.
3. Migrate saved selections once to Mission Control/default and emit one
   deterministic notice.
4. An explicit `mode drunken_master` command returns a local retirement message
   and replacement suggestions; it never reaches the LLM.
5. Historical records retain their identifier and render a `LEGACY MODE` badge.
6. Document that this explicit human decision supersedes the earlier
   deprecation-but-selectable invariant; preserve the prior decision history.

Bobby Hill retirement:

1. Remove `bobby_hill` from selectable web/TUI catalogues, completion, and
   active-roster counts in the same migration release.
2. Preserve immutable legacy metadata and historical rendering; it carries no
   executable persona prompt after retirement.
3. Migrate saved selections once to Full Troll and emit one deterministic
   notice.
4. An explicit legacy command returns a local retirement message identifying
   Full Troll as the successor; it never reaches the LLM.
5. Expand Full Troll into an original sarcastic-sidekick archetype: skeptical,
   irreverent, willing to puncture weak assumptions, but never permitted to
   ridicule the operator, obscure evidence, or turn confidence into fact.
6. Draw inspiration from the broad dramatic function of sarcastic sidekicks,
   hecklers, and irreverent technical partners, not from the dialogue, likeness,
   signature mannerisms, or trade dress of any one existing character.

Archetype consolidation migration:

1. Replace the Chuck Norris and Bruce Lee selections with one canonical
   `sensei` mode; migrate both saved selections deterministically.
2. Replace Columbo and Deckard with one canonical `detective` mode and add
   Sherlock Holmes as an influence, not a separate selectable mode.
3. Rename HAL 9000 to canonical `the_computer`, with the legacy identifier
   retained as a migration alias.
4. Replace Trinity with canonical `m4tr1x`, with the legacy identifier retained
   as a migration alias.
5. Rename Neuromancer to canonical `the_sprawl`, preserving it as a distinct
   mode and retaining the legacy identifier as a migration alias.
6. Historical records keep their original identifiers and display the current
   canonical archetype alongside a `LEGACY MODE` badge.

Exit criteria:

- Drunken Master appears nowhere as a selectable mode.
- Bobby Hill appears nowhere as a selectable mode.
- Existing configuration and historical records remain readable.
- No alcohol-themed replacement is introduced.
- Full Troll has a complete voice contract and repetition budget.

### W-042-07 — Signature cockpit environments

Build reusable geometry families, then map modes onto them. Do not create
fourteen unrelated page implementations. This work is part of v0.4.2, after the
shared semantics, navigation, accessibility, and character contracts are in
place.

First-wave environments:

- **M4TR1X:** code rain outside content; rare session-seeded White Rabbit
  decoration; operator-link animation bound to real probes. Its voice and
  visual vocabulary may draw from the full ensemble of Matrix archetypes
  rather than centering one character.
- **The Sprawl:** perspective grid populated only by real
  artifacts and relationships; constraints appear as neutral barriers.
- **Sensei:** 8-bit side-view arena with real hunt states driving stance,
  approach, hit, block, failure, recovery, flow, and pivot ripples. The owned
  archetype combines disciplined adaptability, cinematic martial-arts wisdom,
  impossible-feat humor, and the contrasting screen energies associated with
  Bruce Lee, Chuck Norris, Jet Li, Chow Yun-fat, Michelle Yeoh, Cynthia
  Rothrock, and Zhang Ziyi characters. Combos never encode confidence.
- **Detective:** case-board and rain-noir geometry, dossier-gap spotlight,
  overlooked-detail callbacks, deductive chains, identity ambiguity, and
  evidence-linked banter. The archetype folds together the analytical functions
  previously represented by Columbo, Deckard, and Sherlock Holmes.
- **The Computer:** central lens as a real activity/fault annunciator, with
  symmetry, low-motion diagnostics, strategic-simulation motifs associated
  with WOPR, and occasional Gibson-style mainframe references.

#### Generative atmospheric music

Add a lightweight procedural soundtrack mapped to the canonical character
contract. Music is atmosphere, not telemetry: it must never imply evidence,
confidence, severity, completion, damage, or source health.

Requirements:

- provide persistent `off / on` and volume controls in both interfaces, with a
  one-keystroke mute action and visible muted state;
- begin muted until the operator explicitly enables sound; web playback starts
  only after a user gesture and complies with browser autoplay restrictions;
- synthesize locally from small owned tone/rhythm definitions using native or
  already-approved project capabilities; require no streaming service, network
  request, account, or downloaded copyrighted recording;
- give each archetype a restrained palette—for example, sparse martial pulse
  for Sensei, brushed noir tension for Detective, cold arpeggiation for The
  Computer, falling digital ambience for M4TR1X, and deep grid drones for The
  Sprawl—without recreating recognizable film scores or melodies;
- allow only gentle, deterministic changes from broad real states such as idle,
  investigating, discovery, caution, and complete; the same state transition
  must produce the same musical rule change within a seeded session;
- debounce transitions and cap intensity so rapid probe events do not create
  noise, fatigue, or a false sense of urgency;
- immediately duck or pause beneath critical audible alerts and never compete
  with screen-reader output;
- persist the operator's preference locally per workspace while providing a
  global emergency mute;
- when audio output is unavailable, show an honest unavailable state and lose
  no operational meaning or control.

The TUI may control a local audio engine, but terminal rendering and input must
remain independent of that engine. Audio startup, failure, or shutdown may not
block the event loop or leave orphaned playback processes.

Follow-on geometry families may cover Mission Control, Silent Operator,
Strategist, Bureaucrat, and Full Troll after the shared semantic components
prove stable.

Exit criteria:

- Each retained mode changes at least three of geometry, motion language,
  instrument vocabulary, interaction feedback, and narrative voice; color does
  not count.
- State-bound motion is deterministic from real state.
- Random decoration uses a reproducible session seed and never enters evidence,
  logs, event ordering, or test outcomes.
- Evidence/provenance components retain recognizable semantics in every world.
- Character lines are labeled narration/advice, never evidence.
- Full Troll, Sensei, and Detective banter obey per-event and per-session
  repetition limits.

### W-042-08 — QA matrix, analyst study, and release closeout

Automated gates:

- lifecycle ordering, cancellation, partial failure, reconnect, and replay;
- evidence/detail schema and redaction;
- real TUI key/mouse/PTY interactions;
- persisted dossier reopen and workspace switching;
- keyboard navigation and automated accessibility;
- reduced-motion behavior;
- music defaults muted, requires explicit activation, persists preference, and
  stops cleanly;
- seeded music-state transitions are reproducible and audio failures do not
  block either interface;
- mute, volume, alert ducking, and screen-reader suppression behavior;
- theme geometry snapshots by family;
- retired-mode migration and historical rendering;
- version-consistency and built-artifact checks.

Manual scenarios:

- idle and first-run orientation;
- planning and long-running probes;
- first evidence and corroboration;
- contradictory evidence;
- dossier partial/fill transition;
- one provider failure amid successful probes;
- model synthesis and token reporting;
- stop, focus, add, skip, retry, and cancel;
- alert → detail → provenance → return;
- mouse, laptop trackpad, keyboard-only, and screen reader;
- compact laptop and wide workstation layouts;
- every retained character at full, reduced, and disabled effects.
- every first-wave character with music off and on, including rapid state
  changes, critical alerts, workspace switching, and application shutdown;

Study measures:

- time to notice important evidence;
- time to explain a finding's provenance;
- evidence/inference classification mistakes;
- navigation and return-position failures;
- false “is it frozen?” reports;
- character-line repetition annoyance;
- successful task completion without mouse input.

Release gates:

- complete Python suite;
- focused real-PTY interaction suite;
- production web build and type check;
- npm registry-signature, attestation, and vulnerability checks;
- built wheel/sdist installation smoke test;
- rendered browser and terminal walkthrough;
- documentation/help/version parity;
- no unreviewed samples, recognizable score melodies, or network audio
  dependencies;
- no unresolved P0/P1 finding from this plan.

## Character roster recommendation

### Retain as product functions

- Mission Control/default
- Silent Operator/ninja
- Full Troll/hype operator
- Bureaucrat/compliance clerk
- Strategist
- Sensei
- Detective
- The Computer
- The Sprawl
- M4TR1X

### Retire or consolidate

- **Drunken Master:** retire in v0.4.2 as specified above.
- **Bobby Hill:** retire in v0.4.2 and migrate active selections to Full Troll.

### Durable identity migration

This is a product-risk recommendation, not legal advice. Named celebrities,
fictional characters, signature quotations, vehicle names, franchise titles,
and close trade dress create avoidable intellectual-property and product
durability concerns. Original archetypes preserve the desired energy while
allowing Pivotglass to own its worlds.

The product owner has approved four consolidated identities:

| Approved identity | Legacy selections migrated into it | Core analytical posture |
|---|---|---|
| **Sensei** | `chuck_norris`, `bruce_lee` | disciplined adaptation, decisive action, flow, and deadpan impossible competence |
| **Detective** | `columbo`, `deckard`; new Sherlock influence | patient skepticism, deduction, overlooked detail, and ambiguity |
| **The Computer** | `hal9000`; WOPR and Gibson influences | calm computation, simulation, system tension, and machine-scale visibility |
| **M4TR1X** | `trinity`; broader Matrix ensemble influence | reality-testing, operator awakening, connection, and decisive resistance |
| **The Sprawl** | `neuromancer` | streetwise cyberspace navigation, depth, and adversarial infrastructure relationships |

Each consolidated identity receives one canonical mode ID. Legacy identifiers
remain local migration aliases for one documented window and historical records
retain their original identifiers.

Dialogue policy:

- Maintain a reviewed registry for every sourced quotation with speaker,
  character/work, wording, trigger, length, and rights-review status.
- Use only brief, attributed quotations that have been explicitly approved for
  inclusion; the default writing path is original dialogue and transformative
  paraphrase capturing a broad archetypal function.
- Sensei's source palette may include Bruce Lee, Chuck Norris, Jet Li, Chow
  Yun-fat, Michelle Yeoh, Cynthia Rothrock, and Zhang Ziyi and their screen
  characters without imitating any performer's likeness or a single character's
  distinctive voice.
- Detective may use approved brief quotations and original banter inspired by
  the distinct investigative methods associated with Columbo, Deckard, and
  Sherlock Holmes.
- The Computer may make sparing WOPR and Gibson references alongside its
  machine-intelligence voice; M4TR1X may reference its broader ensemble.
- Sourced lines are labeled character narration, never evidence or system
  status, and obey per-event and per-session repetition budgets.
- Do not ship actor likenesses, franchise logos, copyrighted screenshots,
  recognizable music, long quotation banks, or extended verbatim dialogue.

## Resolved decisions

Resolved by the product owner:

1. The consolidated identities Sensei, Detective, The Computer, and M4TR1X are
   approved for v0.4.2.
2. Bobby Hill will retire; Full Troll inherits the broader original
   sarcastic-sidekick function.
3. Signature character environments and approved original archetype identities
   target v0.4.2.
4. TUI session history will be complete and support built-in drill-down and
   exact return behavior, using Claude Code, ELM, and Lynx as interaction
   references.
5. The Sprawl remains a separate, first-class v0.4.2 archetype and environment;
   it is not folded into The Computer or M4TR1X.

All character-name and first-wave environment gates are now resolved.

## Recommended release cut

The v0.4.2 scope is W-042-01 through W-042-08, including the approved original
identities and signature environment first wave. Any individual environment
that cannot meet the shared truthfulness, accessibility, determinism, and QA
gates must be disabled rather than shipped as misleading spectacle.

Implementation order is intentionally dependency-driven:

```text
truthful lifecycle
    → evidence references and drill-down
    → attention and recovery
    → proven TUI navigation + accessible pane navigation
    → character contract + retirement migrations
    → approved-identity migration + signature environments
    → v0.4.2 QA matrix and release closeout
```
